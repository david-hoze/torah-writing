# הוספת מקורות – מתודולוגיה

## עקרונות

1. **אין להמציא טקסט** – כל ציטוט חייב להיות העתקה מדויקת מהספרים הדיגיטליים (`books/ברסלב/ליקוטי מוהרן.md` ו-`books/ברסלב/ליקוטי הלכות.md`).
2. **סיכומים מותרים** – בהערות השוליים של `article-ai.md` מותר לכתוב סיכום קצר שמפנה להערה מלאה ב-`sources-ai.md`.
3. **דילוג בציטוט** – להשתמש ב-`...` או `....` כדי לדלג על קטעים בתוך ציטוט מקורי, בהתאם למוסכמה הקיימת.
4. **blockquote (`>`) לטקסט מילולי בלבד** – שורות שמתחילות ב-`>` חייבות להכיל טקסט שהועתק מהספר. טקסט עריכתי (הסבר, קישור בין מקורות) נכתב ללא `>`.
5. **bold (`**`)** – לסימון ביטויים מפתח בתוך הציטוט שרלוונטיים לטענה במאמר.
6. **מקף עברי** – להשתמש במקף עברי (מַקָּף, U+05BE) כמו בספרים המקוריים.

## מקורות מותרים

- ליקוטי מוהר"ן (LM) – `books/ברסלב/ליקוטי מוהרן.md`
- ליקוטי הלכות (LH) – `books/ברסלב/ליקוטי הלכות.md`

## כלי חיפוש

```bash
node scripts/hebrew-search.mjs "<מילות מפתח>" "books/ברסלב/ליקוטי מוהרן.md" --context 3
node scripts/hebrew-search.mjs "<מילות מפתח>" "books/ברסלב/ליקוטי הלכות.md" --context 3
```

- החיפוש מפשיט ניקוד ומטפל באותיות סופיות אוטומטית.
- אפשר להשתמש ב-wildcard (`*`) ובחיפוש proximity (`--proximity N`) למילים מרוחקות.

## תהליך עבודה

1. **לבדוק חומר קיים** – לפני חיפוש, לוודא שההערה לא קיימת כבר ב-`sources.md`.
2. **לזהות טענה** – לסמן את הטענה במאמר שצריכה מקור.
3. **לחלץ מילות מפתח** – לבחור מילות מפתח ייחודיות מהטענה (שם תורה, הלכה, ביטוי ייחודי).
4. **לחפש** – להריץ חיפוש ב-LM/LH.
5. **לקרוא הקשר** – לקרוא את הטקסט סביב התוצאה ולהבין את עומק המקור.
6. **להעתיק** – להעתיק את הטקסט הרלוונטי מהספר הדיגיטלי, ולהדביק ב-`sources-ai.md`.
7. **להוסיף הערה** – להוסיף סימון `[^N]` ב-`article-ai.md` ליד הטענה.

## מבנה הקבצים

### `sources-ai.md`

```markdown
## הערה N

### שם ההלכה/התורה + אות

טקסט עריכתי (אופציונלי, ללא >)

>טקסט מילולי מהספר הדיגיטלי
>עם **הדגשות** של מילות מפתח
```

### `article-ai.md`

העתק מדויק של `article.md` עם תוספת סימוני `[^N]` בלבד (ללא שינוי טקסט המאמר).

## אימות

לאחר הוספת מקורות, להריץ:

```bash
node scripts/verify-citations.mjs sources-ai.md
```
